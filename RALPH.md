# Ralph in Cursor

Ralph - техника для автономной разработки с AI. Агент выбирает задачи из PRD, выполняет их по одной, коммитит и обновляет прогресс.

## Структура файлов

```
pesiki-bot/
├── PRD.md              # Документ с требованиями (задачи в виде чекбоксов)
├── progress.txt        # Лог выполненных задач
├── RALPH.md            # Эта инструкция
└── .cursor/
    ├── commands/
    │   ├── ralph.md        # Одна итерация Ralph
    │   ├── ralph-plan.md   # Создание/редактирование PRD
    │   └── ralph-status.md # Проверка статуса
    └── rules/
        └── ralph.mdc       # Правила для методологии Ralph
```

## Как использовать

### 1. Создай план

Запусти команду `/ralph-plan` и опиши что хочешь построить. AI создаст PRD.md с задачами разбитыми на чекбоксы.

### 2. Запусти итерацию

Используй `/ralph` для выполнения одной задачи. AI:
- Прочитает PRD и progress.txt
- Выберет следующую незавершённую задачу
- Реализует её
- Сделает коммит
- Обновит progress.txt

### 3. Повтори

Запускай `/ralph` снова и снова. Каждая итерация = один коммит = одна задача.

### 4. Проверяй статус

Используй `/ralph-status` чтобы увидеть прогресс.

## Режимы работы

### Human-in-the-loop (рекомендуется)

1. Запускаешь `/ralph`
2. Смотришь что AI сделал
3. Ревьюишь коммит
4. Запускаешь снова

### Semi-autonomous

В Cursor можно использовать Agent Mode и просто попросить:
```
Run 5 Ralph iterations
```

AI выполнит несколько итераций подряд.

## Отличия от Claude Code

| Claude Code | Cursor |
|-------------|--------|
| `./ralph-once.sh` | `/ralph` команда |
| `./afk-ralph.sh 20` | "Run N Ralph iterations" |
| `--permission-mode acceptEdits` | Agent Mode |
| `-p` (print mode) | Нет прямого аналога |

## Советы

1. **Маленькие задачи** - Задача должна быть выполнима за одну итерацию
2. **Чёткий PRD** - Чем яснее PRD, тем лучше результат
3. **Частый ревью** - Проверяй каждый коммит, особенно вначале
4. **Git важен** - Ralph работает через коммиты, инициализируй git если нет
